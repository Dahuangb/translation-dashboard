# 数学复习乐园 - 自动化测试说明

## 📦 测试框架

本项目使用 **Vitest** 作为自动化测试框架，配合：
- **Vitest**: 快速、轻量级的测试运行器
- **jsdom**: 模拟浏览器环境
- **@testing-library/jest-dom**: DOM 断言工具

---

## 🚀 运行测试

### 安装依赖
```bash
npm install -D vitest jsdom @testing-library/jest-dom @vitest/coverage-v8
```

### 运行测试

#### 1. 交互模式（开发时用）
```bash
npm run test
```
- 自动监听文件变化并运行相关测试

#### 2. 单次运行
```bash
npm run test:run
```
- 运行所有测试并退出

#### 3. 生成覆盖率报告
```bash
npm run test:coverage
```

## 🧪 测试文件说明

### 1. `src/utils/progressLogic.ts`
这是所有进度逻辑的纯函数库，包含：
- 题目计数、完成状态检查、进度计算、题目完成操作

### 2. `src/test/progressLogic.test.ts`
包含 **20+ 个测试用例，覆盖以下场景：

| 测试类别 | 测试项数 | 说明 |
|----------|----------|------|
| 基础数据 | 2 | 数据加载和初始进度 |
| 计数函数 | 3 | 各层级统计函数 |
| 状态检查 | 4 | 完成状态和完成率 |
| 完成题目 | 3 | 单题和多题完成 |
| 完整知识点 | 2 | 完成整个知识点 |
| 统计一致性 | 2 | 各统计函数一致 |
| 边界条件 | 3 | 无效ID和边界情况 |
| 集成测试 | 1 | 完整用户学习流程模拟 |

## 📊 测试覆盖范围

### ✅ 已覆盖的功能：
1. **数据验证**
   - 知识点和单元数据正确加载
   - 初始状态正确初始化

2. **计数逻辑**
   - 总题目数
   - 已完成题目数
   - 知识点题目统计
   - 单元题目统计

3. **状态检查**
   - 知识点完成检查
   - 单元完成检查
   - 完成率计算

4. **题目完成**
   - 单题完成
   - 重复完成同一题不重复给星
   - 完成率变化

5. **知识点完成**
   - 所有题目完成
   - 正确给星计算
   - 完成详情记录

6. **统计一致性**
   - 知识点统计与总统计一致
   - 单元统计包含知识点统计
   - 完成率计算一致

7. **边界条件**
   - 无效知识点ID
   - 无效单元ID
   - 无效题目ID

## 📝 添加新测试

### 添加新的测试文件

```typescript
import { describe, it, expect } from 'vitest';
import { yourFunction } from '../your-file';

describe('功能描述', () => {
  it('应该做什么', () => {
    // 准备
    const input = '...';
    
    // 执行
    const result = yourFunction(input);
    
    // 断言
    expect(result).toBe(expected);
  });
});
```

## 🎯 测试最佳实践

1. **每个测试独立**
   - 独立运行
   - 使用 `beforeEach` 重置状态

2. **描述清晰**
   - 用中文描述测试目的

3. **覆盖边界**
   - 正常情况
   - 边界情况
   - 错误情况

4. **纯函数优先**
   - 优先测试纯函数
   - 易测试、易维护

## 🐛 调试测试失败

1. **查看详细输出**
```bash
npm run test:run -- --reporter=verbose
```

2. **只运行特定测试**
```typescript
it.only('只运行这个测试', () => {
  // ...
});
```

3. **跳过某个测试**
```typescript
it.skip('跳过这个测试', () => {
  // ...
});
```
